\textbf{Key finding}: The ceml/ceml sorry (C1) is NOT an engineering gap — it is a mathematical
obstruction. The statement $\forall x > 0,\; (\texttt{ceml}\;t_1'\;t_2').\texttt{eval}\;\uparrow x
\in \texttt{slitPlane}$ is \emph{false} (counterexample:
$\texttt{ceml}(\texttt{const}\;0)(\texttt{const}\;e) \equiv 0 \notin \texttt{slitPlane}$).
The correct response is to reformulate the theorem, not to try to prove the sorry.

\section{Ranked Open Items}

\subsection{\textcolor{high}{Priority 1 — \texttt{sin\_not\_in\_eml\_depth0} [30 min, 0 sorry]}}

\textbf{Statement}: No depth-0 EML tree represents $\sin$.

\textbf{Why first}: Zero effort. Depth-0 trees are constants. $\sin(0) = 0 \neq 1 = \sin(\pi/2)$,
so $\sin$ is non-constant and cannot be represented. Immediate Lean proof by \texttt{linarith}.

\textbf{Impact}: Closes depth-0 case of sin impossibility. Lifts sorry count at no cost.

---

\subsection{\textcolor{high}{Priority 2 — Depth-2 sin impossibility [2–4h, no new math]}}

\textbf{Statement}: No depth-$\leq 2$ real EML tree represents $\sin$.

\textbf{Strategy}: For depth-2 trees $\texttt{ceml}(t_1, t_2)$ where $t_1, t_2$ have depth $\leq 1$:
\begin{itemize}
  \item On any interval $[a,b]$ avoiding zeros of $t_2$, both $t_1$ and $\log|t_2|$ are analytic.
  \item Their difference $(t_1 - \log \circ |t_2|)$ is analytic, so has finitely many zeros by
        \texttt{analytic\_finite\_zeros\_compact}.
  \item Taking $N \to \infty$ intervals gives a bound.
\end{itemize}

\textbf{Difficulty}: Formalizing $\log \circ |f|$ for analytic $f$ with finitely many zeros.
Requires tracking zero-free intervals of $t_2$ (which are finite in number by T91).

\textbf{Impact}: Closes depth-2 case. Gives a pattern that may generalize to depth $k$ inductively.

\subsection{\textcolor{med}{Priority 3 — CONJ\_BOUNDARY\_DECIDABLE}}

\textbf{Statement}: SC + EDB $\Rightarrow$ ELC-membership is decidable.

\textbf{Status}: The EDB positive-domain part ($B^+(n) = n$) is conceptually proved.
General EDB requires the quantitative zero bound $Z(n)$ for EML trees on $\mathbb{R}$,
which is the same obstruction as C2 (o-minimal structure).

\textbf{Impact}: Landmark theorem. Requires C2 first.

\subsection{\textcolor{med}{Priority 4 — Reformulate eml\_tree\_analytic}}

\textbf{Statement (reformulated)}: Every real EML tree is analytic on $(0,\infty)$ \emph{restricted
to the domain where all log arguments are positive}.

\textbf{Why needed}: The current sorry is mathematically correct to be there.
The right theorem tracks a domain predicate $D(t, x)$ through the tree structure:
\[
  D(\texttt{ceml}(t_1, t_2), x) \;:\iff\; D(t_1, x) \;\wedge\; D(t_2, x) \;\wedge\; t_2.\texttt{evalReal}(x) > 0.
\]
On this domain, every subtree's log argument is positive, so the slit-plane sorry evaporates.

\textbf{Lean sketch}:
\begin{verbatim}
def EMLTree.positiveDomain : EMLTree → Set ℝ
  | .const c  => Set.univ   -- constant, no log
  | .var      => Set.univ   -- variable, no log
  | .ceml t1 t2 =>
    t1.positiveDomain ∩ t2.positiveDomain ∩ {x | 0 < t2.evalReal x}

lemma eml_tree_analytic_pos_domain (t : EMLTree) :
    AnalyticOnNhd ℝ t.evalReal t.positiveDomain
\end{verbatim}

\textbf{Effort}: 2–3h Lean.  \textbf{Impact}: Gives the right theorem for the EDB application.

\section{Concrete Attack Plan: Priority 1 (30 min)}

Add to \texttt{InfiniteZerosBarrier.lean}:

\begin{verbatim}
-- sin cannot be represented by a depth-0 EML tree
theorem sin_not_in_eml_depth0 :
    ∀ t : EMLTree, t.depth = 0 →
      ¬ (∀ x : ℝ, t.evalReal x = Real.sin x) := by
  intro t ht hsin
  cases t with
  | const c =>
    have h0 := hsin 0
    have hpi := hsin (Real.pi / 2)
    simp [EMLTree.evalReal, EMLTree.eval,
          Real.sin_zero, Real.sin_pi_div_two] at h0 hpi
    linarith
  | var =>
    have h0 := hsin 0
    have hpi := hsin (Real.pi / 2)
    simp [EMLTree.evalReal, EMLTree.eval,
          Real.sin_zero, Real.sin_pi_div_two] at h0 hpi
    linarith [Real.pi_pos]
  | ceml _ _ => simp [EMLTree.depth] at ht
\end{verbatim}

This is sorry-free and would bring PROVED\_COUNT to 25.
