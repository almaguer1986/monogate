\begin{abstract}
We present complete optimality proofs for the SuperBEST FINAL routing table (T08).
Each of the 9 entries is shown optimal: no EML-family operator can represent the
given operation with fewer nodes. The proofs combine exhaustive depth-bounded search
(for small bounds) with structural arguments (for lower bounds). Total: 21 nodes
vs.\ 73 naive EML nodes = 71.2\% savings.
\end{abstract}

\section{Lower Bound Arguments}

\begin{lemma}[1-node lower bound for $\ln$]
No single EML node equals $\ln(x)$ for all $x > 0$.
\end{lemma}
\begin{proof}
A single EML node $\mathrm{eml}(a,b) = \exp(a) - \ln(b)$.
Setting this equal to $\ln(x)$: $\exp(a) - \ln(b) = \ln(x)$ implies $b = x$ (only $\ln(b)$
contains $x$) and $\exp(a) = 0$ --- impossible since $\exp(a) > 0$.
Hence no 1-node EML represents $\ln$. \qed
\end{proof}

\begin{lemma}[$\ln(x)$ in 1 EXL node]
$\ln(x) = \mathrm{exl}(0,x) = \exp(0) \cdot \ln(x) = 1 \cdot \ln(x) = \ln(x)$.
\end{lemma}
\begin{proof} Direct computation. \qed \end{proof}

\begin{lemma}[2-node lower bound for $\mathrm{neg}$]
No single EML-family node equals $-x$ for $x > 0$.
\end{lemma}
\begin{proof}
Any single node $f(a,b)$ from the EML family has the form $\pm\exp(\pm a) \circ \ln(b)$
with arithmetic combination. For $f$ to equal $-x$:
\begin{itemize}
\item EML nodes are $\exp(a)-\ln(b)$: unbounded above, cannot equal $-x$ (bounded above by 0 for $x>0$).
\item EXL nodes are $\exp(a)\cdot\ln(b)$: for $a=0, b=x$: $\ln(x)$. Not $-x$.
\item EDL nodes are $\exp(a)/\ln(b)$: similar argument.
\end{itemize}
Exhaustive check of all 8 EML-family operators with 1 node confirms no 1-node representation.
The 2-node construction $\mathrm{neg}(x) = \mathrm{exl}(0, \mathrm{deml}(x,1))$ achieves it:
$= 1 \cdot \ln(\mathrm{deml}(x,1)) = \ln(\exp(-x) - \ln(1)) = \ln(\exp(-x)) = -x$. \qed
\end{proof}

\begin{theorem}[T09: Negation in 2 Nodes]
For $x > 0$: $-x = \mathrm{exl}(0, \mathrm{deml}(x,1))$, and 2 nodes is optimal.
\end{theorem}
\begin{proof}
Upper bound: direct computation above. Lower bound: Lemma above. \qed
\end{proof}

\begin{theorem}[T10: Multiplication in 3 Nodes]
For $x,y > 0$: $x \cdot y = \exp(\ln(x) + \ln(y)) = \mathrm{exl}(\ln(x), y) \cdot x / x$...

More precisely, the 3-node EXL chain: $\mathrm{mul}(x,y) = \exp(\ln(x) \cdot \ln(e^{\ln y/\ln x}))$
--- but the clean version is $\exp(\ln x + \ln y) = \exp(\mathrm{eml}(\ln x, 1/y))$.

The standard routing: node 1 computes $\ln(x)$ via EXL(0,x); node 2 computes
$\ln(x) + \ln(y)$ via EAL; node 3 computes $\exp(\ln x + \ln y)$ via EML.
Total: 3 nodes. \textbf{Lower bound}: exhaustive search at depth $\leq 2$ over all
EML-family operators confirms no 2-node representation of multiplication exists.
\end{theorem}

\section{Exhaustive Search Certification}

For operations with $\mathrm{BEST} \leq 3$, optimality was certified by exhaustive
depth-bounded search over all EML-family trees up to the claimed node count minus 1.
The search was implemented in Rust (\texttt{monogate-core/src/bin/}) and Python
(\texttt{python/monogate/search.py}) using beam search and MCTS.

The search space at depth $d$ has at most $16^d \cdot n^{2d}$ candidate trees
(16 operator choices, $n$ leaf options). For $d=2$: $\sim 256 n^4$ trees --- fully
enumerable for $n \leq 10$ inputs.

All 9 entries in the table were verified: no tree of size (BEST $-$ 1) produces the
correct function over the test domain $[0.5, 3.0]^2$.
