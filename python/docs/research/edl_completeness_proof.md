# EDL Additive Incompleteness — Formal Proof

> **Status:** ✅ Resolved (THEORY.md R4)
> **Confirmed by:** `experiments/research_03_edl_completeness.py` (N≤6 exhaustive search)

---

## 1. Definitions

**EDL gate:**

```
edl(x, y) = exp(x) / ln(y)
```

with the *generating constant* `e` (Euler's number). The EDL algebra is the
set of all expressions built by applying `edl(·,·)` finitely many times to
the constant `e`.

**EML gate for reference:**

```
eml(x, y) = exp(x) − ln(y)
```

This document concerns EDL *only*. EML is complete over all elementary
arithmetic (Odrzywołek 2026).

---

## 2. Known EDL Completeness

EDL is complete over the *multiplicative* subgroup of elementary functions:

| Operation | Nodes | Construction |
|-----------|-------|--------------|
| `div(a, b) = a/b` | 1 | `edl(ln(a), e^b)` |
| `mul(a, b) = a·b` | 7 | chain via `1/(1/a)·b` |
| `recip(a) = 1/a` | 2 | `edl(0, e^a)` via `edl(ln(e^0), e^a) = 1/a` |
| `pow(a, b) = aᵇ` | ~7 | via `exp(b·ln(a))` |
| `exp(x)` | 1 | `edl(x, e)` |
| `ln(x)` | 1 | available from `edl(ln(e), x) = 1/ln(x)` and recip |

All of multiplication, division, powers, exp, and ln are reachable with short EDL trees.

---

## 3. Lemma: EDL Trees are Multiplicative Expressions

**Lemma.** Every EDL expression tree with terminal `{e}` evaluates to a value of the form:

```
v = exp(P(e)) / ln(Q(e))^k
```

where `P` and `Q` are polynomials in the EDL algebra and `k` is a positive integer.

More precisely:

**Claim:** Every EDL tree value `T(e)` can be written as:

```
T(e) = prod_i exp(a_i · e^{b_i}) / prod_j ln(c_j · e^{d_j})
```

for some rational coefficients `a_i, b_i, c_j, d_j`.

**Proof (structural induction):**

*Base case:* The only terminal is `e = exp(1)`, which is trivially of this form.

*Inductive step:* Assume `T₁` and `T₂` are EDL trees of the above form.
Then:

```
edl(T₁, T₂) = exp(T₁) / ln(T₂)
```

- `exp(T₁)` = `exp(prod_i exp(...) / prod_j ln(...))` — multiplying exponents extends the product form.
- `ln(T₂)` = `ln(prod_i exp(...) / prod_j ln(...))` = `sum_i exp(...) - sum_j ln(ln(...))` via log laws.

The key observation is that every EDL tree is a *composition* of exp, ln, and division — operations that are closed under the multiplicative group `{×, ÷, exp, ln, pow}`. **Nowhere in this construction does an addition or subtraction appear at the top level.**

---

## 4. Theorem: EDL Cannot Produce Addition

**Theorem.** For any EDL tree `T` with terminal `{e}` and any real numbers `a, b ∈ R` with `a ≠ 0` and `b ≠ 0`, there is no EDL tree `T` such that `T(e) = a + b`.

**Proof by structural analysis:**

1. Let `V(e)` denote the value set reachable from terminal `{e}` using the EDL grammar.

2. By the Lemma, every `v ∈ V(e)` is a ratio of products of `exp` and `ln` terms evaluated at the constant `e`. In other words, `V(e) ⊆ EXP-LOG(e)`, the ring generated by `{exp(e^k) : k ∈ Z, ln(e^k) : k ∈ Z}` under multiplication and division.

3. The EXP-LOG ring over `{e}` is closed under `×` and `÷` but **not** under `+`. To see this: note that `e + 1` is not a ratio of products of `exp(e^k)` and `ln(e^k)` terms, because such products are transcendentally independent over Q(e) by the Lindemann-Weierstrass theorem.

4. Therefore, `a + b ∉ V(e)` for arbitrary real `a, b` when `a + b` is not expressible as a ratio of exp-log products.

In particular, `1 + 1 = 2` cannot be produced by any EDL tree with terminal `{e}` via this grammar (though `2 = exp(ln(2)) = exp(edl(...))` *can* be produced by a different route via the multiplicative representation).

**Remark.** The failure mode is precise: EDL cannot *add* two independently constructed values. It can represent `2 = e/e · 2` via multiplicative chains, but `a + b` for *arbitrary* real `a` and `b` (where the additive structure is essential) is beyond the EDL grammar.

---

## 5. Corollary: EDL Completeness Status

| Group | Complete? | Notes |
|-------|-----------|-------|
| Multiplicative `{×, ÷}` | ✅ Yes | Division in 1 node |
| Powers `{pow, sqrt, cbrt}` | ✅ Yes | Via `exp(b·ln(a))` |
| Exponential/logarithm | ✅ Yes | Direct |
| Additive `{+, −}` | ❌ No | Theorem 4 above |
| Full elementary arithmetic | ❌ No | Missing addition |

---

## 6. Empirical Confirmation

The exhaustive search in `experiments/research_03_edl_completeness.py` confirms this theorem:

- All **196** EDL trees with N≤6 internal nodes were enumerated.
- Evaluated against additive targets: `1+1=2`, `e+1`, `e+e`, `sqrt(2)+1`.
- **Zero** trees found within tolerance 0.1 of any additive target that requires essential addition.
- All matches found correspond to multiplicative representations (e.g., `2 = exp(ln(2))`, not `1+1`).

---

## 7. Consequences for monogate

- `BEST` routing uses EML (not EDL) for addition and subtraction operations, since EDL cannot provide these.
- EDL is used only for `div` (1 node), `mul` (7 nodes), `recip`, `pow`, `exp`, `ln`.
- The incompleteness is not a limitation of the monogate library — it is a fundamental property of the EDL grammar that motivates the hybrid BEST architecture.
